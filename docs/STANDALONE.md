# Standalone Автопостинг — интеграция и валидация

Форк `avito-project` под ТЗ: один оператор управляет N магазинами Avito.
Документ для (а) запуска/проверки и (б) последующей интеграции в панель владельца.

## Что сделано (по фазам)

| Фаза | Содержание |
| --- | --- |
| 0 | Каркас: 1-операторский логин (env), middleware без ролей/paywall, навигация |
| 1 | Схема: `products.city`, `avito_items.orders_*`, кэш баланса/рейтинга сессии, `avito_media_presets`, `avito_post_jobs`, `avito_promotion_daily`, bucket `avito-presets`, сид-каталог |
| 2 | Дашборд по ТЗ: 8 KPI (аванс, ср.расход/день нед., активные, просмотры/избранное/контакты/заказы за мес, рейтинг), блок AI-агента (входящие/ответы/конверсия), окно статистики по заказам (вместо воронки) |
| 3 | Объявления: per-item стата `(+сегодня)` + «заказали», вкл/выкл/удаление через браузер, привязка к товару |
| 4 | Автопостинг: товар→название(генерация)→цена→выложить, рандом-метро кольца, пресеты/микс/уникализатор (sharp), обложки Nano Banana, описание GPT, BullMQ-воркер через stealth-браузер |
| 5 | Заказы с Avito: связка с объявой, трек, цена с комиссиями, статус, код возврата, тег `source_tag='avito'` для страницы панели |
| 6 | AI-реле: фото→vision, голос→Whisper→GPT, ответ текст/(+опц. озвучка TTS) обратно |

## Что требует проверки на живом аккаунте (`// STUB: verify selectors`)

- `src/lib/avito/posting.ts` — селекторы формы «Разместить объявление»
  (категория/атрибуты зависят от товара; форма многошаговая).
- `src/lib/avito/item-actions.ts` — кнопки снять/активировать/удалить в кабинете.
- `src/app/api/avito/webhook/route.ts` — структура `content.voice` у Avito.

Каркас флоу реальный (stealth-браузер + fingerprint/прокси сессии = антидетект
на каждый магазин); требуется выверить CSS-селекторы под текущую вёрстку Avito.

### Цикл выверки селекторов (один заход)

При любом сбое постинга/действий воркер сохраняет **скриншот + HTML** страницы
в `AVITO_DEBUG_DIR` (по умолчанию `<tmp>/avito-debug`) и пишет путь в лог
(`[avito-debug] <label>: screenshot=... html=... url=...`). Лейблы:
`post-listing-title|submit|error`, `item-activate|deactivate|delete|*-error`.

Порядок: деплой → подключить тестовый магазин (прокси + SMS) → одна выкладка →
если упало, забрать из `AVITO_DEBUG_DIR` png+html и лог ошибки → передать
разработчику для точечной правки селекторов в `posting.ts`/`item-actions.ts`.
Парсинг (синки) при этом работает сам — он не через UI-форму.

## Валидация (Node в среде сборки не было — проверить у себя)

```bash
cp .env.example .env.local        # заполнить Supabase/Redis/OPERATOR_*; ключи AI опционально
npm install                       # добавлен sharp
npm run db:migrate                # применяет миграции 20260517000001..3
npm run db:gen-types              # ВАЖНО: регенерит database.generated.ts с новыми
                                  # таблицами → строгая типизация (loose-клиент остаётся
                                  # совместимым, но типы станут точными)
npm run lint                      # ESLint
npm run build                     # сборка + проверка типов
npm run dev                       # http://localhost:3000  (вход: OPERATOR_LOGIN/PASSWORD)
npm run worker:dev                # отдельный процесс: постинг/синк/AI/действия
```

Смоук-чек: вход → дашборд (KPI/AI/заказы) → подключить магазин (сессия Avito,
нужен прокси в `avito_proxies`) → синхронизация → объявления (стата/цена/вкл-выкл)
→ «Создать»: загрузить пресеты, выбрать товар, сгенерировать название, выложить →
статус заявки в «Последние выкладки» → воркер логирует шаги.

Примечание про типы: до `db:gen-types` новые таблицы доступны через
`createServiceClientLoose()` + ручные типы (`src/types/database.ts`). После
регенерации они появятся в `database.generated.ts`; loose-клиент менять не
обязательно (он остаётся валидным).

## Антидетект на каждый магазин

Без изменений сохранён механизм исходного проекта: уникальный
`browser_fingerprint` на аккаунт, 1 прокси = 1 сессия (`avito_proxies`),
human-timing/джиттер, шафл сессий. Новый код (постинг, действия над
объявлениями) переиспользует тот же `openAvitoSessionBrowser`
(`src/lib/avito/browser.ts`) — те же fingerprint+cookies+прокси сессии.
